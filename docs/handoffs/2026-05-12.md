### C. Architecture — email-bridge (SPEC §6.9, ADR-003)

Big revision in PR #8 rework. PR #1's design used catch-all + To: local-part. Replaced by:

1. **Single inbox** — all newsletters subscribed at `inbox@socialisn.com`. Catch-all removed; spam at random local-parts now drops.
2. **List-Id is the canonical sender identifier**, with fallback chain `list_id → from_addr → from_domain`. Substack publishers share `substack.com` as a From: domain — List-Id disambiguates them.
3. **Two Workers, shared D1**: `email-worker/` (writer, Email Routing handler) + `feed-worker/` (reader, route `inbox.socialisn.com/feeds/*`). Independent scaling + deploy + blast-radius.
4. **D1 schema is four tables**: `inbox`, `inbox_links` (FK→inbox CASCADE; replaces TEXT-encoded JSON `links` column), `sender_map` (`(match_field, match_value)` → slug), `unmatched` (operator triage queue).
5. **Adding a new newsletter** — subscribe at `inbox@`; first email goes to `unmatched`; one `sender_map` row + one `sources` row registers the source; Phase 1 PR 4 re-processes earlier `unmatched` rows.

### E. Implementation defaults

- **UUIDv4 for seed rows, UUIDv7 for app inserts.** SPEC §5 amended in 006 to carve this out. The v7 time-ordering rationale doesn't apply to seed data.
- **`cost_ledger` / `backfill_run` columns defined ad-hoc** in `001_init.sql` since SPEC §12/§13 left them informal. Worth canonicalising in SPEC later.
- **CHECK constraints over PG ENUM** for all SPEC §5 enum-like columns. Cheaper to evolve.
- **`noUncheckedIndexedAccess: true`** kept — catches real bugs.
- **`fileParallelism: false`** kept — cheaper than schema-per-test isolation.
