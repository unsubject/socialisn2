## Decisions made and why

1. **Self-hosted GH Actions runner on srv1565522, not SSH from cloud.** Hostinger firewall ("Gandalf", id 293921) locks port 22 to a single source IP. Three options on the table: (a) GH IP allowlist (~1500 ranges, rotating; broad surface), (b) dynamic per-deploy firewall rule via Hostinger API (race-prone, orphaned-rule risk), (c) self-hosted runner polling outbound. Picked (c): one-time setup, eliminates the bug class, simplifies workflow by ~65 lines, no inbound exposure. Trade-off in `docs/deploy.md`: anyone with push to `main` can execute on the VPS — fine for solo, harden ACL (required reviews on `.github/workflows/**`) before adding collaborators.

2. **Traefik wiring matches existing Odoo deployment pattern.** Read the live `odoo-ff0q` `docker-compose.yml` on the VPS to confirm: Traefik runs in `network_mode: host` from its own compose project `traefik-yffq`, discovers containers via Docker socket, routes via container bridge IP from the host's network namespace. NO shared external network. Cert resolver named `letsencrypt` (the memory `hostinger_traefik_cf_pattern` had said this; PR #68 still ended up with `mytlschallenge` — corrected in #72).

3. **`rootDir="."` in `tsconfig.build.json`.** Multi-input (`src` + `scripts` + `config`) forces project-root rootDir. Output shifts to `dist/src/`, `dist/scripts/`, `dist/config/`. Path-shift cost: Dockerfile CMD + three docker-compose service commands. Considered moving `config/` under `src/` to keep `rootDir=src/` (smaller diff), but kept project-root structure because `config/` has non-TS content (prompt templates, `litellm.yaml`) that fits a "static config" dir better than under `src/`. Deferred as a refactor candidate for the Phase 5 PR 4 phase report.

4. **`post-deploy-backfill` aborts on BOTH `unreachable` AND `not_configured`.** Silent shipping of misconfigured prod (Stage 5 `archive_overlap=0` every run) is a worse failure mode than a red deploy. `::error::` annotation in the workflow log on either.

5. **Hex passwords (`openssl rand -hex 32`) for `POSTGRES_PASSWORD`.** Initial base64 password broke URL parsing in `DATABASE_URL`. Memory `postgres_url_encoded_password.md` saved.

6. **No `traefik.docker.network` label, no `networks:` block on app/nginx.** With Traefik in host mode discovering via socket, the auto-detect path works as long as the container is on a single network — `<project>_default` is fine.
