## Decisions made and why

1. **Scoped token covers exactly 4 permissions** — Workers Scripts:Edit (account), D1:Edit (account), Workers Routes:Edit (zone scoped to `socialisn.com` only), Account Settings:Read. Matches CF's standard "Edit Cloudflare Workers" template plus D1. Rejected alternatives: (a) including DNS:Edit on `socialisn.com` — verified unnecessary, the deploy-workers run confirmed Custom Domain reconciliation on `inbox.socialisn.com` does not require it; (b) including Email Routing — Email Routing rules are configured out-of-band per email-worker/README.md, CI never touches them.

2. **Retired `bootstrap-d1.yml` rather than expanding the scoped token to allow `wrangler d1 create`** — the workflow's only job is creating the D1 database, which already exists (`socialisn2-inbox`, uuid `ab7605c4-be15-4499-ac42-e1d8202f9b51`). Keeping it would either require broadening the token (defeating the structural fix) or leave a workflow that silently fails on next dispatch. If a future bootstrap is needed: do it from a local `wrangler login`-authenticated machine, not from CI.

3. **Kept the GHA secret name `CLOUDFLARE_API_TOKEN`** (did not rename to `CF_API_TOKEN_SOCIALISN2`). Trade-off accepted: per-token boundary is not visible in YAML, but the diff is minimal and the convention matches CF/wrangler defaults. Document the scoping out-of-band (this handoff + build task notes).

4. **Used `gh secret set` (CLI hidden prompt) for the rotation transport** instead of pasting the token in chat. Established as the new default in `~/.claude/team-lead-protocol.md`: when MCP can't move a credential, the preference ladder is CLI hidden-prompt → destination web UI → chat-paste (last resort only). This rule is now part of the global protocol; the lesson came from a near-miss during this session.
