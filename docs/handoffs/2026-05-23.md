## Decisions made and why

1. **Each project gets its OWN scoped CF API token.** The 2nd-brain
   deploy-mcp-worker was wedged for hours because socialisn2's morning
   token rotation revoked the account-wide token that 2nd-brain's GHA
   secret also pointed at. Fixed by issuing a 2nd-brain-specific scoped
   token (`Workers Scripts:Edit` + `Hyperdrive:Edit` only — no
   `Account Settings:Read` needed when `CLOUDFLARE_ACCOUNT_ID` is passed
   via env). Forward rule: any new CF-deploying project gets its own
   scoped token, no shared tokens across repos.

2. **Gemini paid-tier upgrade is required for socialisn2 operation.**
   Free tier caps at 20 req/min. Phase 2's `SCORING_WORKER_BATCH_SIZE=20`
   fires every minute, so the 21st+ item in each batch 429'd. With the
   16K backlog, ~85% of attempted items poisoned into `raw_items_failed_3x`.
   Upgrade unblocks throughput; 429-backoff (PR #90) is defense-in-depth.

3. **`docker compose restart` ≠ `docker compose up -d` for env_file
   changes.** Restart signals stop+start to the existing container,
   keeping the baked-in env from creation time. Env changes require
   `up -d --force-recreate` (or just `up -d` when image SHA also
   changed). This bit hard during the Telegram bot debugging — fixing
   `TELEGRAM_CHAT_ID` in `.env` did nothing until `up -d --force-recreate
   app` ran.

4. **`docker compose build` (no service arg) is required when worker
   images share the Dockerfile.** Each service with `build: .` produces
   its own image tag. PR #85 fixed deploy-vps.yml to do this.

5. **Two reusable ops workflows added.** `test-run-now.yml` and
   `dump-logs.yml` close the gap between "I need to verify VPS-side state"
   and "I don't have SSH." Both run on the self-hosted runner and surface
   output to job summary + stdout. Use via `gh workflow run` from
   anywhere with repo-write.

6. **`TWO_BRAIN_MCP_TOKEN` must be in lockstep with 2nd-brain's
   `BRAIN_MCP_TOKEN` Worker secret.** Drift causes silent Stage 5
   degradation (archive_overlap = 0 on every run, no curation novelty
   signal). Set after rotating either side; verified via the
   post-deploy-backfill assertion (now passing).

- **`docker compose build` per-service caveat** — `build app` only
  rebuilds the app image; each `build: .` service has its own tag.
- **`docker compose restart` doesn't re-read env_file** — use
  `up -d --force-recreate` for env changes.

  in 3.x; remove from all call sites if bumping.

  breaks structural assignment to plain function types; wrap mocks in
  an intersection-type wrapper for the contract.
- **BullMQ 5.x rejects `:` in queue names** — reserves it for the
  Redis key-prefix separator.
- **drizzle-kit 0.28 / vitest 4 / vite 8 + esbuild peer-dep tangle**
  — needs npm `overrides: { esbuild: ^0.28.0 }` to force-dedupe.
- **CF scoped-token pattern**: prefer per-project tokens (Workers
  Scripts + bindings-as-needed) over account-wide ones. Rotating an
  account-wide token cascades across all projects sharing it.
