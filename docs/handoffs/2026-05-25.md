## Decisions made and why

1. **The threshold hypothesis (from the 2026-05-23 handoff) was wrong.**
   `config/domains.ts` doesn't exist on main, so there was no per-domain
   threshold to "omit," and 0.30 is *looser* than spec floors (would make
   fewer clusters, not more). The real bug was the `last_seen_at` semantics.
   Confirmed empirically: 12,413/12,570 clusters born stale.

2. **The `items.authority_weighted` "stale snapshot gap" was a false alarm.**
   A mid-session diagnosis worried that ranking preferred a frozen snapshot
   over recalibrated source authority. But `authority_weighted` is NULL on
   all 16,404 items (never written by the ingestion INSERT), so the
   `COALESCE(i.authority_weighted, s.authority_score)` ALWAYS uses live
   source authority. Recalibration reaches ranking. **No code fix needed for
   Track B** — it had simply never been exercised (`feedback = 0` ever).

3. **Did NOT recompact the ~12,400 historical orphan clusters.** They have
   `last_seen_at = old publishedAt`, so they're already outside the 7-day
   window that BOTH ranking and `compactClusters` use → never candidates,
   zero cost (Stage 3 only scores in-window clusters), they age out. And
   `compactClusters` is O(N²)/domain (designed for <100), so a mass pass
   over ~2,500/domain is a heavy query + 12k-row rewrite for no candidate
   benefit. Heal-forward + the daily compaction cron on the in-window set is
   the right path. (Confirmed via reading `compactClusters` in full.)

4. **Ran the validation now via the UTC budget reset, not by raising the
   ceiling.** The day rolled over mid-session ($1.50 → fresh), so no prod
   `.env` surgery was needed — the `set-cost-ceiling` workflow stayed parked.

## Cost-ceiling timing (important)

`cost_ledger` sums spend per **UTC day** (`date_trunc('day', NOW(), 'UTC')`),
ceiling from `COST_CEILING_DAILY_USD` (default $1.50), read per-call (so a
change needs `up -d --force-recreate`, not restart). A run that exhausts the
day's budget makes all subsequent runs return 0 / `cost_ceiling_hit` until
00:00 UTC. The clustering fix should drop per-run cost as the active window
heals (fewer clusters advancing to Stage 4).
