## How the audit ran

A `Workflow` orchestration: **7 parallel review lenses** (aggregation math, SQL/DB
path, MCP contract, Telegram/MarkdownV2, orchestrator integration, test adequacy,
cost/perf/security) → each finding **adversarially verified** by a skeptic agent to
filter false positives. 18 agents total. Result: **11 candidate findings → 5
confirmed, 6 dropped.** Then the fix/CI loop was driven from the main session
(clone → fix → tsc/lint/test → push → watch CI → merge → deploy).

The verification layer was the value-add: it killed 6 false positives (hypothetical
microsecond-tie races, a misread of a deliberate empty-headline guard, a
"redundant escaping" claim impossible for any valid enum input) and *corrected* two
survivors.

## Decisions made and why

1. **Fixed the corrupted-URL bug (should-fix), with a smarter escaper than the
   agent proposed.** `formatCandidateDetail` ran the full `escapeMarkdownV2` inside
   the `(url)` of a MarkdownV2 link — backslashes before `.` etc. are retained
   literally, corrupting nearly every `/cand` source link. The agent's suggested
   raw-passthrough was itself buggy (a `)` in a URL closes the link early), so added
   `escapeMarkdownV2Url` escaping ONLY `)` and `\`. (Pre-existed the trending work;
   fixed because the audit surfaced it and it's near-universal.)
2. **Fixed two test-integrity gaps:** the ≥1 theme-floor test used a count-2 tag
   (proved nothing) → use a real single-cluster theme; dedup tests asserted only
   `cluster_count` → added a `top_headline` assertion locking "latest row wins".
3. **Removed a dead `domains` column fetch** in `computeTrending` (aggregation keys
   on `primaryDomain` only).
4. **Did NOT add DI for the degrade-not-block catch** (the one confirmed nit left
   unfixed). Making `computeTrending` injectable to test the catch would break the
   file's convention — its peer scoring fns (`computeTemperature`/`Trajectory`/
   `Exclusive`/…) are all directly imported, not in `RunDependencies`. The verifier
   agreed; not worth a convention break for a logic-free catch.
5. **Left the PG dedup test's headline assertion off** — without controlling
   `created_at` it'd risk a timestamp-tie flake; the pure test locks that contract
   deterministically via input order.

## Tried and abandoned
